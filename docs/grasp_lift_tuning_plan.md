# Захват і підйом чашки: розрахунки та експерименти

Повний опис знахідок і поточних параметрів: **`docs/grasp_tuning_findings.md`**.

---

## Що було зламано (і виправлено)

### A. Гриппер (сила затиску)

- **Підйом перестав спрацьовувати** після зміни фінального затиску з **GRIPPER_CLOSED (0.0)** на **GRIPPER_GRASP_MUG (0.02)**.
- При 0.02 гриппер залишався трохи відкритим; у симуляції це давало слабший контакт — чашка не трималась при підйомі.
- **Виправлення:** у `close_gripper_top` знову використовується **GRIPPER_CLOSED (0.0)** після пре-закриття 0.018.

### B. Торсо не опускався в descend_vertical (критичний баг)

- При вході в `descend_vertical` перевірялось: якщо `(tz - mz) <= target_gap` (0.06), торсо одразу **заморожувалось**.
- dZ вже було ~0.055, тому торсо ніколи не опускалось, tool лишався на 5.5 см вище чашки, гриппер закривався «в повітрі».
- **Виправлення:** прибрано ранню freeze; у `descend_vertical` завжди викликається опускання торсо до 0.0. Перехід у `close_gripper_top` тільки по умові dZ (без xy в descend).

## Як далі відпрацьовувати без «вгадувань»

### 1. Один параметр за раз

- Міняти тільки **один** параметр у серії прогонів (наприклад, тільки `approach_clearance` або тільки значення закриття гриппера).
- Фіксувати в таблиці: значення параметра, `grasp_success`, `grasp_lift_delta_m`, `grasp_retry_count`.
- Порівнювати з baseline (прогон з поточними дефолтами).

### 2. Мінімальний набір розрахунків

- **Зупинка перед чашкою:** tool_x_stop = mug_x − approach_clearance. Поточний baseline: **0.13 м**.
- **Висота над чашкою при закритті гриппера:** перехід у `close_gripper_top` коли dZ = tool_z − mug_z ≤ **top_descend_clearance = 0.025 м**. Торсо має реально опускатись (не freeze на вході в descend_vertical).
- **Гриппер:** 0 = повне закриття, ~0.045 = повне відкриття. Для утримання при підйомі — **0.0**.

### 3. Експерименти по черзі

| Експеримент | Параметр | Діапазон / значення | Що дивитись |
|-------------|----------|----------------------|-------------|
| A. Відстань зупинки | approach_clearance | 0.10, 0.13, 0.16, 0.20 м | Чи не під’їжджає занадто близько (чашка не штовхається), grasp_success, lift_delta |
| B. Сила затиску | значення в close_gripper_top (після 0.018) | 0.0 (повне закриття) vs 0.01 vs 0.02 | grasp_lift_delta_m, чи чашка випадає при підйомі |
| C. Час утримання перед verify | кроки в close_gripper_top перед переходом у verify_grasp | 270 vs 360 | Чи встигає фізика «осісти», стабільність підйому |
| D. Висота над чашкою при спуску | top_descend_clearance | **0.025** (baseline), 0.03, 0.04 м | Чи гриппер опускається достатньо низько; перехід тільки по dZ |

### 4. Що логувати для аналізу

- У кожному кадрі вже є: tool (x,y,z), mug (x,y,z), mug_tilt_deg, gripper opening (з joint positions).
- Корисно додати в звіт або physics_log на момент verify: `gripper_opening_at_verify`, `mug_z_at_verify` — щоб бачити, при якому затиску і висоті чашки проходить verify і чи піднімається чашка далі.

### 5. Критерій «працює»

- `grasp_success == True`: підйом **≥ 0.02 м** (`grasp_lift_delta_m`), `mug_final_z <= mug_z0 + 0.05`. Обмеження на tilt прибрано (див. `grasp_tuning_findings.md`).
- Стабільність: кілька прогонів підряд з однаковими параметрами — скільки success.

## Підсумок

- **Гриппер:** повне закриття **GRIPPER_CLOSED (0.0)** для top-grasp.
- **Descend:** торсо завжди опускається в `descend_vertical`; перехід у `close_gripper_top` тільки по dZ.
- **Поточні baseline:** `config/grasp_tuning.json`, повний опис — `docs/grasp_tuning_findings.md`.
- **Далі:** змінювати по одному параметру, фіксувати результати в таблицю.
