# Холодильник у сцені grasp

У режимі `--grasp` до сцени додається **процедурний холодильник** з відкриваною дверима, ручкою та полками всередині.

## Що є в сцені

- **Позиція:** центр холодильника в світі `(FRIDGE_X, FRIDGE_Y) = (2.8, 0)` м — за столом (стіл у `x=2.0`).
- **Розміри:** ширина (Y) 0.8 м, глибина (X) 0.7 м, висота 1.6 м.
- **Дверцята:** вертикальна вісь обертання (RevoluteJoint, axis Y). Петлі зліва (від’ємний Y), дверцята відкриваються вправо (позитивний Y). Максимальний кут відкриття 90°.
- **Ручка:** на правій стороні двері (з боку робота), брус 6×12×3 см — робот може фізично схопити ручку і відкрити дверцята тягою.
- **Полки:** 4 полки всередині кабінету (колізія), на них можна класти предмети.

## Фізика

- **Кабінет:** kinematic rigid body (нерухомий).
- **Дверцята:** dynamic rigid body (маса 8 кг), з’єднані з кабінетом через `UsdPhysics.RevoluteJoint`. Drive на шарнір не ставиться — дверцята відкриваються лише від зовнішньої сили (наприклад, робот тягне за ручку).
- Ручка — дочірній прим двері, тому при відкритті рухається разом із нею.

## Керування з CLI / run_bench.ps1

- За замовчуванням у режимі `--grasp` холодильник **додається** (еквівалент `--fridge`).
- Щоб не додавати холодильник: `--no-fridge` (Python) або `-NoFridge` (run_bench.ps1).
- Явно увімкнути: `--fridge` / `-Fridge`.

Приклад:

```powershell
.\scripts\run_bench.ps1 -Grasp -NoVideo -NoFridge   # сцена без холодильника
.\scripts\run_bench.ps1 -Grasp -Fridge              # з холодильником (за замовчуванням)
```

## Модель: процедурна vs ассет

Зараз використовується **процедурна модель** (куби/бокси в `test_robot_bench.py`, функція `_spawn_procedural_refrigerator`). Це дає:

- гарантовану колізію та шарнір;
- незалежність від Nucleus/інтернету;
- ручку, придатну для захвату роботом.

Якщо потрібна модель з **Isaac Sim / Omniverse Nucleus** або з інтернету:

1. Завантажити USD холодильника (наприклад, через Asset Browser у Isaac Sim або з репозиторію 3D-моделей).
2. Зберегти USD у проєкт (наприклад, `C:\RoboLab_Data\data\fridge.usd`).
3. У коді можна додати гілку: якщо існує файл ассету — `add_reference_to_stage(fridge_usd, "/World/Fridge")`, і на ассет навісити шарнір і ручку через USD; інакше — викликати `_spawn_procedural_refrigerator` як зараз.

Константи сцени (позиція, розміри) оголошені у `scripts/test_robot_bench.py`: `FRIDGE_X`, `FRIDGE_Y`, `FRIDGE_WIDTH_Y`, `FRIDGE_DEPTH_X`, `FRIDGE_HEIGHT`, `FRIDGE_DOOR_OPEN_DEG`.
